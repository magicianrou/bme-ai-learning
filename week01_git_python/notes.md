# Week 01 Notes: Git, GitHub, and Python Basics

## 1. What I learned today

Today I learned the basic workflow of Git and GitHub:

1. Create a local project folder.
2. Initialize a Git repository.
3. Add files to the staging area.
4. Commit changes locally.
5. Connect the local repository to GitHub.
6. Push local commits to the remote repository.

## 2. Important Git commands

git status
git add .
git commit -m "message"
git push

## 3. My understanding

Git is used to manage different versions of a project.
GitHub is used to store the project online.

A commit is like a saved checkpoint of the project.
A push uploads local commits to GitHub.

## 4. Problems I encountered

When I first tried to push the repository, Git rejected the push because the remote repository already had a README file.
I solved this by merging the remote branch and resolving the README.md conflict.
